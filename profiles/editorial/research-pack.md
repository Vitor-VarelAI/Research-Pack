# Research pack

Regra principal:

```txt
Primeiro recolher sinais.
Depois confirmar chão.
Depois encontrar a jogada.
Só no fim escrever.
```

O `research pack` não é artigo. É matéria-prima organizada para diagnóstico editorial.

Não deve tentar soar bonito. Deve separar sinal, fonte, facto, inferência e hipótese.

## Ordem

```md
# Research Pack

## Sinal
O que chamou a atenção.

## Fontes
Official / jornalística / técnica / policy / mercado / comunidade.

## Claims confirmados
Factos que aparecem em fontes fortes.

## Claims frágeis
Coisas que parecem verdade mas ainda precisam de confirmação.

## Contradições
O detalhe estranho.

## Timing
Porque isto pode estar a acontecer agora.

## Incentivo
O que a empresa ou actor quer mudar.

## Distribuição
Onde isto vive: API, app, cloud, marketplace, OS, social, hardware.

## Dinheiro
Como isto pode virar receita, lock-in ou controlo.

## Impacto no workflow
O que muda para designers, devs, criadores, filmmakers, VFX, fotografia.

## Ângulos possíveis
3-5 leituras editoriais.

## Grau de confiança
Facto confirmado / inferência provável / hipótese editorial / especulação fraca.
```

## Classificação de fontes

Usa pelo menos 3 links úteis antes do diagnóstico:

1. **Official**: empresa, docs, changelog, API docs, modelo, help center, policy.
2. **Jornalística**: Reuters, TechCrunch, The Verge, MIT Technology Review, FT, NYT, etc.
3. **Técnica / policy / mercado**: paper, benchmark, GitHub, regulator, SEC/FTC/EU, pricing, terms, stock/earnings.

Se o tema envolver privacidade, copyright, segurança, legal, finanças, benchmarks ou superlativos, procurar uma 4.ª fonte.

## Separação obrigatória

Nunca misturar estes níveis:

### Facto confirmado
Algo directamente suportado por uma fonte forte.

Exemplo:

```txt
Reuters reporta que autoridades chinesas discutiram limites com Alibaba, ByteDance e Z.ai.
```

### Inferência provável
Leitura razoável a partir de factos confirmados, mas não declarada pela fonte.

Exemplo:

```txt
Se modelos chineses baratos forem restringidos, startups que dependem deles podem ter custos mais altos.
```

### Hipótese editorial
Ângulo que vale explorar, mas ainda é tese.

Exemplo:

```txt
A China pode estar a passar de open models como estratégia de adopção para open models como alavanca de poder.
```

### Especulação fraca
Ideia interessante mas sem chão suficiente. Pode entrar como pergunta, não como afirmação.

Exemplo:

```txt
Isto pode ser preparação para uma política formal de export controls sobre modelos.
```

## Claims frágeis

Marca como frágil quando:

- só aparece numa fonte secundária
- vem de marketing da empresa
- depende de benchmark sem validação independente
- usa linguagem vaga: “sources say”, “may”, “could”, “reportedly”
- envolve intenção, estratégia, geopolítica ou mercado sem prova directa

## Output útil

O research pack deve permitir responder rapidamente:

- o que sabemos?
- o que ainda não sabemos?
- onde está a contradição?
- qual é a jogada provável?
- vale escrever sobre isto?
